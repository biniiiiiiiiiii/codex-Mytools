一、第 1 轮 PRD
1.1 项目名称

JSON 工具站（自用版）

1.2 第 1 轮目标

在最短时间内交付一个可用的纯前端 MVP，满足你日常对 JSON 的两个核心需求：

格式化 / 压缩

校验 / 错误提示

同时把后续第 2、3 轮的扩展位预留好，但本轮不做复杂功能。

1.3 产品定位

这是一个给自己高频使用的工具站，不追求“大而全”，追求：

打开就能用

不依赖后端

数据尽量不离开浏览器

页面简单，操作直接

为后续 Diff、JSONPath、C# 转换器留扩展空间

1.5 目标用户

你自己，典型场景：

调接口时复制一段 JSON 进来快速格式化

接口报错时快速定位 JSON 语法问题

临时把 JSON 压缩成单行粘贴到请求参数或日志里

切页后不想丢失刚刚处理过的内容

1.6 用户故事
用户故事 1

作为开发者，我想把一段混乱的 JSON 一键格式化，方便阅读和排查字段。

用户故事 2

作为开发者，我想把一段 JSON 压缩成单行，方便放到配置、请求体或日志里。

用户故事 3

作为开发者，我想在 JSON 非法时立刻看到报错位置和可理解的提示，而不是只看到一串异常。

用户故事 4

作为开发者，我想随时复制处理后的结果，而不是自己手选全文。

用户故事 5

作为开发者，我想刷新页面后还能保留刚才输入的内容。

1.7 功能需求
A. JSON 输入区

功能：

支持粘贴 JSON

支持编辑

支持大段文本滚动

支持示例数据填充

支持清空

规则：

默认可以输入任意文本

不强制实时格式化

本轮不做拖拽文件导入

B. 格式化

功能：

解析输入内容

将合法 JSON 输出为格式化后的字符串

默认 2 空格缩进

异常：

如果输入非法 JSON，不生成结果

结果区显示为空或保留上次合法结果，二选一
我建议：本轮直接清空结果区并显示错误

C. 压缩

功能：

将合法 JSON 压缩成单行字符串

不改变语义

异常：

输入非法时不给结果，提示错误

D. 校验

功能：

点击“校验”后解析 JSON

成功则显示“JSON 合法”

失败则显示：

错误摘要

尝试提取行号、列号

友好解释文案

提示文案示例：

可能缺少逗号

可能多了尾随逗号

属性名必须用双引号

字符串没有正确闭合

E. 结果区

功能：

显示格式化结果 / 压缩结果 / 校验结果

支持复制

对成功和失败状态有明显区分

F. 本地持久化

功能：

自动保存最近一次输入

页面重新打开时恢复

本轮只存：

最近输入内容

最近选中的功能页

不存：

多条历史记录

收藏模板

1.8 非功能需求
性能

处理常见接口 JSON 时不应明显卡顿

1MB 左右文本尽量保持可操作

可维护性

功能逻辑与 UI 分离

JSON 工具函数独立放到 shared/utils

可扩展性

预留 features/diff、features/path-query、features/converter

安全性

所有 JSON 处理默认在浏览器本地完成

本轮不向任何服务端发送内容

1.9 验收标准

满足以下条件即可验收：

应用可正常启动

页面有基础导航和布局

能输入 JSON

能格式化合法 JSON

能压缩合法 JSON

能校验 JSON

非法 JSON 有清晰错误提示

可复制结果

可清空输入/输出

刷新后可恢复最近输入

核心工具函数有基础测试

README 可指导本地启动
